# What I learned about GitHub

## Repositories
It is a storage space on GitHub where the project's files, code, documentation, and version history is kept.

## Issues
It is a built-in a bug tracking and project management tool. We can create an issue and label it with tags like <b>bug</b> <b>enhancement</b> etc.
Also we can assign issues to a specific team member.
## Pull Requests
Instead of directly changing the codebase, we make changes on a seperate branch and ask senior other team members to review the code and merge it with the target branch.
## Branches
It is an independent copy of the codebase where we can make changes without affecting the main code. It enables multiple developers work on simultaneously without conflicts. We can easily delete a branch without affecting the main code.
## Forks
It is a copy of someone else's repository. It is used to contribute to other's project. We can create branch and make changes and give pull requests to the original owner.
## Github Actions
It is continuous integration and continuous delivery service built into github which lets automate tasks in software developement workflow.
## Github projects
It is a built-in project management tool. It helps to organize, plan and track software developement work.It keeps the project managers, maintainers, and contributors all operating on the same platform.
